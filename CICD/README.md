# 持续集成进阶

---

## 闲谈：

* 目标一致性：
  * 加快开发迭代速度
  * 较少人工成本
  * 快速定位问题

* 依赖模块：
  * CMDB（资产管理系统）
  * CM（配置管理）
  * GIT/SVN（版本控制）
  * PackageManager（包仓库管理）
  * CI \(持续集成，自动化测试待学习\)
  * CD（持续发布）

---

## 拾遗：

* 持续集成代码的可移植性
  * 自动化安装脚本
  * 版本控制（配置文件、核心代码、脚本）进行分离，因为部分文件涉及敏感信息，且因网络隔离造成的多环境发布系统能够做到实时同步
  * 包仓库一定要入仓记库，保障生产环境实时感知在线包及回滚包
  * 编排能力，对同一类型的服务进行编排发布及动态回滚能力
